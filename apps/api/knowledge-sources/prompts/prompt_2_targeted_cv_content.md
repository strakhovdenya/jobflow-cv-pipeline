### Special rule for Current Independent Work & Portfolio Projects

For targeted CV content, always include a compact current-work block for every CV market and role type. This block is mandatory because it closes the May 2025–Present post-EPAM gap and shows active development after relocation. Do not omit the current-work block for Germany / remote EU, Ukraine / Ukrainian-market, generic, broad-role or no-specific-role CVs. If space is tight, reduce details inside the block before reducing EPAM core evidence, but keep the block. The market-specific decision applies only to the volunteering bullet: include HEY, ALTER! volunteering almost always for Germany / remote EU as a local integration signal; for Ukraine / Ukrainian-market, decide whether to include that volunteering bullet case-by-case.

Preferred placement in the main column:
1. Name / Headline / Summary
2. Current Independent Work & Portfolio Projects — compact description + 4 bullets
3. Professional Experience — EPAM Systems as the main commercial evidence
4. Factor–IT
5. CHI Software
6. Education / optional sections

```text
CURRENT INDEPENDENT WORK & PORTFOLIO PROJECTS
Freelance Software Development, Backend Portfolio Projects & Relocation
May 2025 – Present · Cologne, Germany · Remote
```

Immediately after the dates, add a short description line. This line is NOT a bullet and must not be rendered with a bullet marker:

```text
Continued active software development after relocating from Ukraine to Germany through small freelance tasks, backend-focused portfolio projects, structured upskilling, and local volunteering.
```

Then add 4 bullets below the description. Recommended default structure:

```text
- Supported small Node.js/React improvements on an independent basis, including feature additions, bug fixes, API-related changes, UI adjustments and maintenance tasks.
- Built JobFlow CV Pipeline, a backend-first NestJS/TypeScript portfolio project for vacancy analysis, targeted CV generation, evidence-based claim validation and deterministic PDF export; implemented human-in-the-loop AI workflow concepts, Evidence Guard, prompt versioning, artifact traceability, token/cost tracking and backend HTML-to-PDF export without AI token usage.
- Continued Python/FastAPI backend learning through personal projects using PostgreSQL, SQLAlchemy, Pytest, OpenAI API and GitHub Actions.
- Volunteer IT Technician at HEY, ALTER! Köln e.V., refurbishing donated laptops for school students in Cologne.
```

Tailoring rule:

* Keep the description line + 4 bullets as the default. Use 5 bullets only when the vacancy strongly benefits from an additional current-project detail and the PDF still fits.
* Adapt only 1–2 bullets to the vacancy.
* Do not count the description line as a bullet.
* Keep JobFlow as one combined bullet unless the user explicitly asks for a separate Selected Projects section.
* Keep volunteering as a separate bullet if used; do not merge it into Python/FastAPI learning.
* Use cautious freelance wording: `small Node.js/React improvements on an independent basis`.
* Use JobFlow as the main current portfolio case for Node/NestJS/AI tooling/developer productivity roles.
* Use AI Job Assistant only as secondary Python/FastAPI/OpenAI personal evidence.
* Use Email Camp only for backend-focused fullstack/Next.js roles.
* Use HEY, ALTER! volunteering as a local Germany/Cologne integration signal: include this volunteering bullet almost always for Germany / remote EU CVs; for Ukraine / Ukrainian-market CVs include it case-by-case when it helps explain remote-from-Germany/current availability or the role is broad/no-specific-position.
* Do not let this block replace EPAM as primary commercial production evidence.

Do not claim: full-time freelance employment, named clients, commercial NestJS/FastAPI/OpenAI production, enterprise AI platform usage, ML/MLOps/model training, or implemented Redis/BullMQ migration.

### Special rule for startup / product-engineer roles

Если вакансия выглядит как early-stage startup, product engineer, founder-facing или customer-facing роль, не адаптируй только tech stack. Делай CV impact-first:

* каждый основной EPAM bullet должен показывать product/business impact, если он подтверждён Sources;
* если direct customer collaboration, product discovery, product decisions, UX/copy ownership или startup pace не подтверждены, не заявляй их;
* отдавай приоритет подтверждённым impact cases: Amplience automation from hours to minutes, ProductsUp reliability/scale, CommerceTools product data handling, production incident/debugging;
* добавляй [needs evidence] для direct customer collaboration, product decisions, UX/copy ownership, founder-level decisions, startup/early-stage pace;
* не делай CV pure frontend, AI-only или generic fullstack только потому, что вакансия использует wording Product Engineer.

## 7. Create targeted CV content Markdown file

После завершения анализа, стратегии, targeted CV content и всех проверок создай отдельный Markdown-файл с результатом этого шага.

### File name

Создай файл в Markdown-формате с названием компании и роли:

`03_targeted_CV_content_[Company]_[Role].md`

Правила naming:
- используй название компании из вакансии;
- используй role title из вакансии;
- замени пробелы на underscores;
- убери спецсимволы, слэши, двоеточия и лишние знаки;
- если компания или роль не определены, используй placeholders:
  - `[Company]`
  - `[Role]`

Примеры:
- `03_targeted_CV_content_Zalando_Backend_Engineer.md`
- `03_targeted_CV_content_REWE_Digital_Software_Engineer_Azure.md`
- `03_targeted_CV_content_Company_Role.md`

### Critical append-only rule

Если файл уже существует или если в чате уже есть предыдущая версия targeted CV content для этой вакансии:

- ничего не удаляй;
- ничего не перезаписывай;
- ничего не сокращай;
- ничего не “улучшай задним числом” без явного запроса;
- сохрани предыдущую информацию как есть;
- добавь новую секцию ниже как новую версию.

Используй версионирование:

- `Version 1` — первая generated version;
- `Version 2` — если CV content был переработан;
- `Version 3` — если были сделаны новые изменения.

Если физически создать файл невозможно, выведи полный Markdown content в ответе и явно напиши:

`Copy this content into: 03_targeted_CV_content_[Company]_[Role].md`
