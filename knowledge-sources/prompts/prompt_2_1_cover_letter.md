## 1. Cover letter strategy

Определи:

* главный angle под эту вакансию;
* 2–3 strongest matches между вакансией и моим опытом;
* какие career cases использовать;
* что лучше не упоминать;
* нужно ли упоминать work authorization Germany;
* нужно ли упоминать Cologne / remote / hybrid;
* есть ли language risk, который не стоит подсвечивать лишний раз.

## 2. Evidence selection

Выбери максимум 3 evidence points из моих Sources.

Приоритетные evidence areas:

* Node.js / TypeScript commercial backend experience;
* Azure Functions / Durable Functions / serverless workflows;
* REST APIs / third-party integrations;
* e-commerce integrations: Amplience, CommerceTools, ProductsUp;
* production debugging / Application Insights / KQL;
* PostgreSQL / SQL / data integrity foundation;
* backend-focused fullstack React/Next.js only if vacancy requires fullstack.
* Current JobFlow / NestJS / AI tooling only if it is clearly relevant, and only as personal portfolio evidence.

Не используй evidence, если оно не связано с вакансией.

## 3. Cover letter draft

Напиши cover letter на английском.

Требования:

* 180–250 слов;
* максимум 4 коротких абзаца;
* professional, direct, human;
* без generic clichés;
* не начинать с “I am writing to express my interest”;
* не писать слишком эмоционально;
* не пересказывать всё CV;
* не делать длинную биографию;
* каждая фраза должна быть связана с вакансией;
* не выдумывать факты о компании;
* если информация о компании есть только в вакансии, используй только её;
* если company/product context unclear, не фантазируй.

Структура письма:

Paragraph 1:
Кто я и на какую роль откликаюсь. Сразу показать релевантность под роль.

Paragraph 2:
2–3 конкретных совпадения с вакансией через мой commercial experience.

Paragraph 3:
Почему мне интересна именно эта роль / продукт / команда, но без выдуманных деталей.

Paragraph 4:
Короткое завершение: готов обсудить, authorized to work in Germany, availability/location if relevant.

* не выглядит ли Python/FastAPI/OpenAI как commercial production;
* не добавлен ли Kubernetes;
* не завышен ли Docker;
* не выглядит ли Terraform/Azure DevOps как DevOps ownership;
* не выглядит ли PHP как current target stack;

* не завышен ли German;
* не завышен ли English;
* не завышен ли ownership по ProductsUp / Amplience / CommerceTools;
* нет ли неподтверждённых метрик.
